### 3. Technical Local SEO Implementation

#### Schema Markup (Implement on ALL pages):

```json
{
  "@context": "https://schema.org",
  "@type": "NewsMediaOrganization",
  "name": "Wellington Times",
  "areaServed": [
    {
      "@type": "City",
      "name": "Wellington",
      "containedIn": {
        "@type": "Country",
        "name": "New Zealand"
      }
    },
    {
      "@type": "AdministrativeArea",
      "name": "Karori",
      "containedIn": {
        "@type": "City",
        "name": "Wellington"
      }
    }
  ],
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Karori",
    "addressRegion": "Wellington",
    "postalCode": "6012",
    "addressCountry": "NZ"
  }
}
```

#### NewsArticle Schema (Add to every article):
- datePublished
- dateModified (update when article is updated)
- author information
- publisher details
- articleSection (Karori, Wellington, Politics, etc.)
- locationCreated: Wellington, NZ

#### NAP Consistency (Name, Address, Phone):
- Add footer with consistent contact information
- Use same format across all pages
- Include in structured data
- Match exactly with Google Business Profile

#### Performance Metrics to Achieve:
- **Largest Contentful Paint (LCP):** < 2.5 seconds
- **First Input Delay (FID):** < 100 milliseconds
- **Cumulative Layout Shift (CLS):** < 0.1
- **Time to Interactive (TTI):** < 3.5 seconds

#### Optimization Checklist:
- [ ] Implement CDN with NZ/Wellington servers (Cloudflare NZ)
- [ ] Compress all images (WebP format, < 100KB each)
- [ ] Minify CSS and JavaScript
- [ ] Enable browser caching (1 year for static assets)
- [ ] Lazy load images below fold
- [ ] Defer non-critical JavaScript
- [ ] Remove unused CSS/JS
- [ ] Use preconnect for external resources
- [ ] Implement critical CSS inline
